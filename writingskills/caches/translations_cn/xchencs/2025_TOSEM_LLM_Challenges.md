# An Empirical Study on Challenges for LLM Application Developers
# LLM应用开发者面临的挑战：一项实证研究

**来源**: TOSEM 2025 | **作者**: Xiang Chen 等

## 摘要
随着大型语言模型（LLM）的快速发展，越来越多的开发者开始构建基于LLM的应用程序。然而，LLM应用开发面临着独特的挑战，这些挑战尚未被系统研究。本文对LLM应用开发者面临的挑战进行了大规模实证研究。我们从Stack Overflow和GitHub Issues中收集了大量与LLM应用开发相关的讨论，通过开放编码和主题分析方法，识别出开发者面临的主要挑战类别。

## 引言核心
- LLM应用开发是一个新兴领域，与传统软件开发有本质区别
- 开发者需要处理prompt工程、模型选择、API调用、成本控制等新问题
- 现有研究主要关注LLM本身的能力，而非使用LLM构建应用时的工程挑战
- 本文旨在填补这一研究空白，为工具和框架设计提供指导

## 研究方法
1. **数据收集**：从Stack Overflow收集LLM相关问题帖子，从GitHub收集LLM框架（LangChain、LlamaIndex等）的Issues
2. **主题分析**：采用开放编码方法，由多名研究者独立编码并达成一致
3. **挑战分类**：将识别出的挑战按主题分组，构建挑战分类体系
4. **定量分析**：统计各类挑战的频率、解决率、响应时间等

## 关键发现
- RQ1：识别出N个主要挑战类别，包括Prompt设计、模型部署、API使用、性能优化、成本控制等
- RQ2：Prompt相关问题最为常见，但模型部署问题最难解决
- RQ3：不同LLM框架的用户面临的挑战分布存在显著差异
- RQ4：提出了针对各类挑战的实践建议和工具需求

## 结论
本文首次系统研究了LLM应用开发者面临的挑战，构建了挑战分类体系，并提出了改进建议。研究发现对LLM工具链和框架的设计具有重要指导意义。
