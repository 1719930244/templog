# Detecting API Compatibility Issues of Android Applications based on Screen Transition
# 基于屏幕转换检测Android应用API兼容性问题

**来源**: EMSE 2026 | **作者**: Xiang Chen 等

## 摘要
Android API的频繁更新导致应用兼容性问题。本文提出基于屏幕转换的API兼容性检测方法。通过分析应用在不同Android版本上的屏幕转换行为差异，自动检测由API变更引起的兼容性问题。实验在多个真实Android应用上验证了方法的有效性。

## 引言核心
- Android API每年更新，开发者难以及时适配所有版本
- 现有静态分析方法误报率高，动态测试方法覆盖率低
- 核心insight：API兼容性问题通常会导致可观察的UI行为差异

## 方法概述
1. **屏幕转换模型构建**：在不同Android版本上运行应用，记录屏幕转换图
2. **差异检测**：比较不同版本间的屏幕转换图，识别异常转换
3. **根因分析**：将异常转换映射到具体的API调用，定位兼容性问题
4. **报告生成**：生成包含问题描述、影响范围和修复建议的报告

## 关键实验发现
- 在多个真实应用上检测到API兼容性问题
- 与现有工具相比，误报率更低，检测覆盖率更高
- 屏幕转换模型能有效捕获UI级别的兼容性异常

## 结论
基于屏幕转换的方法为Android API兼容性检测提供了新视角，在实际应用中表现出良好的检测效果。
